Omega — WiFi-Controlled RC Vehicle

ESP8266 · Embedded C · L298N Motor Driver · Flutter
🟢 Active

Overview

WiFi-controlled robotic vehicle built around the ESP8266 microcontroller with custom embedded motor control logic and a Flutter remote interface for real-time command delivery.

Technical Details

  • Network architecture: ESP8266 hosts a local WiFi access point; Flutter app connects directly and sends UDP command packets
  • Latency handling: UDP chosen over TCP for lower overhead; command rate throttled to prevent buffer overload on the microcontroller
  • Motor driver logic: L298N H-bridge controlled via PWM signals for variable speed and direction on two independent drive channels
  • Control protocol: Directional commands (forward, back, left, right, stop) encoded as single-byte packets for minimal transmission delay
  • Flutter UI: Custom joystick widget with touch event handling and real-time WiFi connection status indicator
Flagship project combining embedded networking, real-time motor control, and cross-platform mobile development into a single integrated system.

NXP Cup — Autonomous RC Car

NXP FRDM Microcontroller · Pixy2 Vision · C/C++
🔵 Completed

Overview

Competition-based autonomous vehicle built for the NXP Cup using onboard vision processing via Pixy2 camera for lane detection and an embedded control algorithm to maintain track position at speed.

Technical Details

  • Lane detection: Pixy2 camera identifies colored track markers; centroid data is fed to steering logic in real time at each control cycle
  • Steering algorithm: Proportional control derived from lateral offset between the detected lane center and vehicle centerline
  • PID loop: Proportional-integral-derivative correction applied to steering output to reduce oscillation and prevent overshoot
  • Real-time constraints: Control loop optimized for fixed-interval execution under the strict timing requirements of the NXP FRDM platform
  • Competition result: Completed all qualifying runs; vehicle successfully navigated the standard track layout
High-value competition project for engineering roles — demonstrates autonomous control, vision integration, PID tuning, and real-time embedded architecture under real constraints.

Parker — 18-Servo Hexapod Robot

Microcontroller · 18× Servo Motors · PWM · C
🟡 Prototype

Overview

Designed and implemented an 18-servo hexapod robot focusing on tripod gait sequencing and synchronized motion control. Built to explore multi-actuator coordination under real embedded hardware resource constraints.

Technical Details

  • Gait algorithm: Tripod gait implemented — alternating groups of three legs lifted and planted in synchronized timing sequences
  • Servo synchronization: PWM signals distributed across 18 channels; timing constraints created coordination challenges at full gait speed
  • Power management: Single power rail caused voltage sag under full servo load, directly affecting motion smoothness and coordination
  • Microcontroller limits: Processing overhead of managing 18 simultaneous PWM channels approached the controller's scheduling capacity
  • Lessons learned: Future iteration would use dedicated servo controller ICs (PCA9685) and distributed power rails for cleaner torque delivery
Movement smoothness was limited due to microcontroller timing constraints and power distribution challenges. This engineering realism demonstrates applied problem diagnosis and iterative design thinking — not a flaw, a finding.

Pallbearer — 2D Unity Platformer

Unity · C# · Physics Engine · Git
🟣 In Development

Overview

Collaborative 2D platformer focused on state-machine-driven character control, physics-based mechanics, and modular level architecture. Role: programming lead — responsible for all gameplay systems and code architecture.

Technical Details

  • State machine: Finite state machine managing player states (idle, run, jump, fall, attack) with clean, decoupled transition logic
  • Collision system: Layered physics collision with platform edge detection and one-way platform pass-through logic
  • Character controller: Custom Rigidbody2D controller for precise physics tuning — chosen over Unity's default CharacterController for greater control
  • Scene architecture: Modular scene loading with persistent game manager using singleton pattern for cross-scene state
  • Version control: Git-based workflow with feature branches enabling parallel art and programming development
State machines, physics simulation, and modular software architecture — skills developed here apply directly to robotic control systems and embedded software design.
